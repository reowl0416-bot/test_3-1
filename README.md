このリポジトリは、現時点では最小構成のテンプレートとして管理されています。
新規参加者向けに、現在の構造と今後の進め方を整理します。

## 1. コードベースの一般的な構造

現在のトップレベル構造は以下のみです。

- `README.md` : プロジェクトの概要と参加者向けの案内を記載するドキュメント

> 補足: ソースコード、設定ファイル、テストディレクトリはまだ作成されていません。

## 2. 知っておく必要がある重要事項

- **このリポジトリは初期状態**です。機能実装前のため、実行手順や依存関係は未定義です。
- **最初に合意すべきもの**
  - 使用言語・ランタイム（例: Node.js / Python / Go）
  - パッケージマネージャ（例: npm / pnpm / pip / poetry）
  - テスト方針（単体・統合・E2E）
  - Lint / Formatter（例: ESLint, Prettier, Ruff, Black）
- **ドキュメント駆動で進めると安全**です。最初に README を更新し、開発ルールやセットアップを明文化すると、参加者が増えても迷いにくくなります。

## 3. 次に学習・整備する事項（推奨ロードマップ）

### Step 1: 開発基盤の決定

1. 技術スタックを決める
2. ディレクトリ構成を決める
3. 最低限の CI（lint/test）を導入する

### Step 2: ひな形の実装

1. エントリーポイント作成
2. サンプル機能を 1 つ実装
3. サンプルテストを 1 つ追加

### Step 3: 開発体験の向上

1. `CONTRIBUTING.md` を追加（ブランチ運用、レビュー方針）
2. `.editorconfig` / formatter 設定を導入
3. Issue / PR テンプレートを導入

## 4. 新規参加者が最初の30分でやること

1. `README.md` を読む
2. チームの技術選定方針を確認する
3. 「最小機能 + 最小テスト」を作る最初のタスクを担当する
